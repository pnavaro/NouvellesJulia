# Octobre 2020

Bonjour à tous, voici quelques liens pour suivre l'actualité du langage Julia.

### Billets de blog

- [Julia and Spark, Better Together](https://juliacomputing.com/blog/2020/06/02/julia-spark.html) Billet sur l'API Julia pour utiliser la très populaire bibliothèque parallèle de manipulation et de traitement de données [Apache Spark](https://spark.apache.org).
- [Javis.jl with Animations.jl](https://opensourc.es/blog/javis-animations/) Un billet de blog sur un package que j'ai mentionné le mois dernier. Javis permet de dessiner des objets mathématiques et il est maintenant possible de faire des animations.
- [How Multiple Dispatch lets you write Composable software](https://mybinder.org/v2/gh/fonsp/vscode-binder/master?urlpath=pluto/open?url=https%3A%2F%2Fraw.githubusercontent.com%2Fmsaroufim%2Fmultiple_dispatch%2Fmaster%2Fsrc%2Fmultiple_dispatch.jl) Une petite démonstration d'un notebook Pluto avec binder.
- [Neural ODE for Differentiable Reinforcement Learning and Optimal Control: Cartpole Problem Revisited](https://medium.com/swlh/neural-ode-for-reinforcement-learning-and-nonlinear-optimal-control-cartpole-problem-revisited-5408018b8d71)
- [Comparing Python and Julia for hydrologic modeling](https://medium.com/@kel.markert/comparing-python-and-julia-for-hydrologic-modeling-7334ffa9534b) : Traduire un code existant de son domaine en Julia, est un excellent moyen d'apprendre le langage. En plus on est rarement déçu par les performances.

### Packages

- [Trixi.jl](https://github.com/trixi-framework/Trixi.jl) Package pour équations différentielles hyperboliques.  Ce travail est intéressant car c'est le genre de problème qu'on peut faire en Python mais on est obligé d'utiliser d'utiliser Cython, numba ou récrire des parties en Fortran ou en C
